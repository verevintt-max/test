изнес-требования для системы управления складом и производством
USER STORY #0: Управление справочником материалов
Как администратор системы
Я хочу управлять базовым справочником материалов
Чтобы иметь единую классификацию для учета и производства

Критерии приемки:

✓ Добавлять материалы в справочник с указанием:

Наименование (обязательно)

Базовая единица измерения (шт, кг, м, л, рулон и т.д.)

Цветовая палитра (опционально, можно выбрать несколько)

Категория (например: "Пластики", "Крепеж", "Краски")

Описание (опционально)

✓ Редактировать существующие записи

✓ Архивировать неиспользуемые материалы (без удаления)

✓ Искать материалы по названию/категории

✓ В справочнике нельзя создать дубликат по "Название+Цвет"

USER STORY #1: Поступление материалов на склад
Как пользователь (мастер/закупщик)
Я хочу вносить материалы в раздел "Склад материалов"
Чтобы учитывать фактическое наличие сырья

Критерии приемки:

✓ Выбирать материал из справочника (US#0)

✓ Автоматически подтягивается единица измерения из справочника

✓ Указывать:

Количество (обязательно)

Дату поступления (обязательно, по умолчанию - сегодня)

Номер партии (опционально)

Цвет (если в справочнике не указан или отличается)

Стоимость за единицу (обязательно)

Место приобретения (ссылка на OZON, WB или просто текст)

Комментарий

✓ Система рассчитывает общую стоимость партии

✓ Просматривать историю поступлений

✓ Нельзя удалить поступление, если материалы уже использованы в изделиях

✓ При редактировании сохраняется история изменений

USER STORY #2: Создание карточки изделия (техкарты)
Как пользователь (технолог/дизайнер)
Я хочу создавать карточки изделий с рецептами
Чтобы стандартизировать производство и расчет себестоимости

Критерии приемки:

✓ Указывать основные параметры:

Название изделия

Категория/теги для поиска

Время производства (в часах/минутах, обязательно)

Ссылка на шаблон/модель (опционально)

Комментарий

✓ Формировать список материалов (BOM):

Выбирать материалы из справочника

Указывать количество на одно изделие

Видеть автоматически подтянутую единицу измерения

Добавлять/удалять позиции из списка

✓ Авторасчет себестоимости при указании:

Себестоимость материалов (берется текущая средняя цена со склада)

Стоимость работы оборудования в час (ставка)

Накладные расходы (% или фиксированные)

Трудозатраты (опционально)

✓ Система показывает:

Себестоимость производства

Рекомендуемую цену с наценкой (настраиваемый %)

✓ Прикреплять файлы:

3D-модели (STL, 3MF)

G-code

Инструкции

Фотографии

Система хранит историю версий файлов

USER STORY #3: Производство готовых изделий
Как пользователь (производственник)
Я хочу из карточки изделия создавать готовые изделия
Чтобы учитывать выпуск продукции и списывать материалы

Критерии приемки:

✓ Выбирать карточку изделия (US#2)

✓ Указывать количество для производства (может быть >=1)

✓ Автоматическое списание материалов:

Система проверяет достаточность остатков

При недостатке - показывает предупреждение

Списание происходит по правилу FIFO (первый пришел - первый ушел)

Для каждой позиции BOM списывается: Количество × Количество_изделий

✓ Созданные изделия попадают в "Склад готовой продукции"

✓ Каждому изделию/партии присваивается:

Статус: "На складе"

Дата производства

Номер партии (автоинкремент или ручной ввод)

Уникальный QR-код для идентификации

✓ Себестоимость изделия фиксируется на момент производства

✓ Можно прикрепить фото готового изделия

USER STORY #4: Отчет по остаткам
Как пользователь (руководитель/логист)
Я хочу видеть актуальные остатки
Чтобы контролировать складские запасы

Критерии приемки:

✓ Виджет "Остатки материалов":

Таблица: Материал | Единица измерения | Остаток | Средняя цена | Общая стоимость

Расчет: Остаток = ΣПоступлений - ΣСписаний

Клик по строке → детализация по партиям

✓ Виджет "Готовая продукция":

Таблица: Изделие | Количество на складе | Дата производства | Себестоимость

Фильтрация по статусам

✓ Общая сводка на дашборде:

Общая стоимость сырья на складе

Общая стоимость готовой продукции

Топ-5 материалов по стоимости

Последние операции

✓ Экспорт отчетов в Excel/PDF

✓ История движений по каждому материалу/изделию

USER STORY #5: Продажа/списание продукции
Как пользователь (менеджер/продавец)
Я хочу фиксировать выбытие готовых изделий
Чтобы остатки всегда были актуальными

Критерии приемки:

✓ Выбирать изделие со склада готовой продукции

✓ Указывать:

Количество

Тип операции: "Продажа", "Списание (брак)", "Возврат поставщику"

Сумма (для продажи)

Клиент (опционально)

Дата

Комментарий

✓ При продаже/списании:

Уменьшается количество на складе

Изделие переходит в статус "Продано" или "Списано"

Фиксируется финансовый результат

✓ При возврате - изделие снова доступно для продажи

✓ Просмотр истории продаж/списаний

USER STORY #6: Просмотр истории операций
Как пользователь (бухгалтер/аудитор)
Я хочу видеть полную историю всех операций
Чтобы вести учет и анализировать деятельность

Критерии приемки:

✓ Единый журнал операций с фильтрами по:

Типу операции (поступление, производство, продажа)

Дате (диапазон)

Материалу/изделию

Пользователю

✓ Для каждой операции отображается:

Дата и время

Пользователь

Тип операции

Объект (материал/изделие)

Количество

Стоимость

Комментарий

Ссылка на связанные документы

✓ Возможность отменить операцию (с пометкой "Отмена")

✓ Все отмены также фиксируются в истории
